## 如何使用

### 方式一：直接转换后上传（推荐）
系统知识库目前支持 **PDF / PPT(PPTX) / Word(DOCX)** 格式上传，Markdown 暂不支持直接解析。你可以：

1. 用 Word / WPS / Typora 等工具打开 `.md` 文件，另存为 `.docx`
2. 或使用命令行工具批量转换（需要安装 [Pandoc](https://pandoc.org/)）：
   ```bash
   pandoc 01_软件工程概述.md -o 01_软件工程概述.docx
   ```
3. 登录教师账号 → 进入「知识库」页面 → 点击「上传文档」→ 选择对应章节 → 上传转换后的 `.docx` 文件

### 方式二：补充自己的课程内容
每份文档都是一份可扩展的模板，建议你：
- 补充学校/教材中的具体案例、习题
- 插入课堂 PPT 的关键图示说明（如数据流图示例、架构图）
- 根据实际授课深度调整内容详略
- 补充章节对应的重点/难点提示，便于后续「章节考核」出题时 AI 能覆盖更精准的知识点

### 配合"章节考核"功能使用
上传知识库文档后，建议同步在「章节管理 → 配置考核」页面，将文档中的关键概念填入"知识点"标签，例如第一章可配置：
```
知识点：软件危机、瀑布模型、敏捷开发、螺旋模型
题型：选择题2 + 判断题2 + 简答题2
```
系统会调用大模型结合这些知识点自动出题，出题质量与知识库内容的完整度、知识点配置的精确度直接相关。
