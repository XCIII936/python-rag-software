# 课程教学智能体系统 — 实现方案

## 一、项目概述

构建一个基于 Vue3 + FastAPI + Qwen3-Max 的课程教学智能体 Web 系统，为《软件工程》课程提供 AI 助教服务。

**核心功能：** 课程问答(RAG)、学习资源推荐、章节考核(自动出题+评价)

---

## 二、技术栈

| 模块 | 技术 |
|------|------|
| 前端 | Vue3 + Element Plus + Pinia + TypeScript |
| 后端 | Python + FastAPI + SQLAlchemy |
| 数据库 | SQLite |
| 向量数据库 | Milvus Standalone (etcd + minio) |
| 大模型 | Qwen3-Max (阿里云 DashScope API) |
| RAG | LangChain + Text Embedding V3 |
| 文档解析 | PyPDF2 + python-pptx + python-docx |
| 通信 | SSE 流式传输 (对话/智能体调用) |
| 认证 | JWT (python-jose + bcrypt) |
| 部署 | Docker Compose (5服务) |

## 四、数据库设计 (SQLite — 12张表 + Milvus)

### 4.1 SQLite 表

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `users` | 用户(学生/教师) | id, username, password_hash, role(student/teacher), email |
| `chapters` | 课程章节 | id, title, description, order_index |
| `knowledge_base_documents` | 知识库文档元数据 | id, chapter_id, file_type(pdf/ppt/docx), status(pending→parsed→failed) |
| `chat_sessions` | 对话会话 | id, user_id, chapter_id(可选上下文) |
| `chat_messages` | 对话消息 | id, session_id, role(user/assistant), content, metadata |
| `chapter_assessment_configs` | 考核配置(教师预设) | id, chapter_id, knowledge_points(JSON), question_types(JSON), evaluation_dimensions(JSON) |
| `assessment_records` | 考核记录 | id, user_id, chapter_id, status(in_progress/completed), total_score |
| `assessment_questions` | 考核题目 | id, record_id, question_type(choice/true_false/short_answer), content, options, correct_answer, user_answer, score |
| `assessment_reports` | 评价报告 | id, record_id, overall_score, dimension_scores(JSON), review_suggestions, strengths, weaknesses |
| `resource_recommendations` | 资源推荐记录 | id, session_id, title, resource_type, relevance_score, source_info |
| `chapter_progress` | 学生章节进度 | id, user_id, chapter_id, status(to_learn/pending/completed) |
| `system_logs` | 系统日志 | id, level, module, action, message, user_id |
| `agent_configs` | 智能体配置 | id, name, system_prompt, model_name, temperature |
| `llm_configs` | 大模型配置 | id, api_key(加密), model_name(qwen3-max), temperature, base_url |

### 4.2 Milvus 集合

**集合名:** `course_document_chunks`
**字段:** id, document_id, chapter_id, content, embedding(1536维, text-embedding-v3), metadata(JSON)
**索引:** HNSW, Inner Product 度量

### SSE 流式事件类型
```
event: data → {"type": "start"}         # 开始
event: data → {"type": "context", ...}  # RAG上下文引用
event: data → {"type": "token", ...}    # 逐字流式内容
event: data → {"type": "recommendation"} # 资源推荐
event: data → {"type": "error", ...}    # 错误信息
event: data → {"type": "done"}          # 完成
```

## 七、三大核心功能数据流

### 1. RAG 课程问答
```
学生提问 → 生成问题向量(Embedding) → Milvus 语义搜索 → 获取相关文档块
→ 构建 Prompt(Qwen3-Max, 含上下文+引用) → SSE 流式返回 → 前端实时渲染 Markdown
```

### 2. 学习资源推荐 (后处理)
```
问答完成后 → 用问题和答案联合查询 Milvus → 排序去重 → 按文档分组
→ 返回 TOP5 推荐(含页码/幻灯片号/视频时间戳) → 前端卡片展示
```

### 3. 章节考核 (最复杂)
```
教师预设: 知识点+题型分布(选择题2+判断题2+简答题2)+评价维度
  ↓
学生开始考核 → LLM出题(JSON格式, temperature=0.3) → 逐题回答(单选/判断/简答)
  ↓
交卷 → 选择/判断自动评分(对比答案) + 简答LLM评分 → 生成评价报告
  ↓
结果展示: 总分(circle进度), 维度评分(雷达图), 建议复习知识点(tag列表)
```

## 八、关键技术设计

### 1. SSE 流式传输
- 管理接口: HTTP JSON (POST/GET)
- 智能体调用: FastAPI StreamingResponse + DashScope SDK stream=True
- Nginx 关闭代理缓冲: proxy_buffering off;
- 前端: fetch + ReadableStream 解析 SSE 事件, Pinia 实时更新

### 2. JWT 认证
- python-jose 签发, 24h 过期
- FastAPI HTTPBearer 依赖注入, 路由层面鉴权
- 双角色: student(学生) / teacher(教师)
- 前端 Axios 拦截器自动携带 Token

### 3. 文件解析
- PDF: PyPDF2 (按页提取文本+页码元数据)
- PPT: python-pptx (按幻灯片提取文本+slide_index)
- Word: python-docx (按段落提取文本+标题层级)
- 中文分块: RecursiveCharacterTextSplitter, 分隔符含 "。" "；" "，"

### 4. 向量存储 (Milvus Standalone)
- DashScope text-embedding-v3 (1536维)
- HNSW 索引, Inner Product 度量
- pymilvus + LangChain Milvus 封装连接
- metadata JSON 存储: page_number, slide_index, chapter_id, file_name

### 5. Docker 多服务编排
```
etcd (Milvus元数据) →  minio (Milvus存储) →  milvus (向量数据库)
  ↓
backend (FastAPI + Uvicorn, :8000)
  ↓
frontend (Nginx静态服务, :8080, 反向代理/api到backend)
```

## 十、验证方案

1. **端到端测试:** 注册→登录→教师创建章节→上传PPT/PDF→学生对话提问(SSE流式显示)→资源推荐显示→参加章节考核→答题→查看评价报告
2. **Docker 验证:** `docker compose up --build` 一键启动, 浏览器访问 `http://localhost:8080`
3. **流式传输验证:** 聊天页面输入问题, 确认答案逐字流式输出, 底部推荐资源卡片显示
4. **考核流程验证:** 教师配置考核参数(知识点+题型+评价维度), 学生开始考核, 完成6题后自动生成评价报告
5. **前后端分离验证:** 前端 Vite 开发服务器 `:5173` 代理到后端 `:8000`
