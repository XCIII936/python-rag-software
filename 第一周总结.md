# 第一周开发总结

> **项目：** 课程教学智能体系统（Course Teaching Agent）
> **技术栈：** Vue3 + Element Plus + TypeScript + Pinia / FastAPI + SQLAlchemy + SQLite + Milvus + DashScope(Qwen3-Max)
> **时间：** 2026年7月 第一周
> **总文件量：** 后端 63 个 Python 文件（~3900行）+ 前端 36 个源文件（~5400行）

---

## 一、✅ 已完成内容

### Phase 1：项目脚手架搭建 ✅

**后端 FastAPI 框架：**
- FastAPI 应用主入口 `app/main.py`，注册了 7 个路由模块
- 数据库引擎配置 `app/db/database.py`（SQLite + SQLAlchemy 2.0）
- 核心配置 `app/core/config.py`（Pydantic Settings，支持 `.env` 读取）
- 工具模块：日志器 `app/utils/logger.py`、文件工具 `app/utils/file_utils.py`

**前端 Vue3 框架：**
- Vite 5 + Vue 3.4 + TypeScript 5.4 开发环境
- Element Plus 2.8 + 自定义 SCSS 变量体系
- Axios HTTP 客户端（请求拦截器自动携带 JWT Token，响应拦截器统一错误处理）
- 前端路由守卫（`permission.ts`，区分学生/教师角色权限）
- 布局组件：`DefaultLayout.vue`（含侧边导航栏）+ `AuthLayout.vue`

### Phase 2：用户管理 + JWT 认证 ✅

**后端实现：**
- **模型：** `User` 表（id, username, password_hash, role: student/teacher, email）
- **接口：**
  - `POST /api/v1/auth/register` — 注册
  - `POST /api/v1/auth/login` — 登录（返回 JWT access_token + user 信息）
  - `GET /api/v1/auth/profile` — 获取个人信息
  - `PUT /api/v1/auth/profile` — 更新个人信息
- **安全：** `passlib + bcrypt` 密码哈希，`python-jose` JWT 签发（24h 过期）
- **依赖注入：** `get_current_user`、`get_current_teacher` 路由层鉴权
- **CRUD 基类：** `app/crud/base.py` 通用 CRUD + 各模块 CRUD

**前端实现：**
- `LoginView.vue` — 登录页
- `RegisterView.vue` — 注册页
- `ProfileView.vue` — 个人信息页
- `stores/auth.ts` — Pinia 状态管理（登录/登出/信息获取）
- `utils/auth.ts` — Token 存储管理
- **路由守卫：** 未登录重定向到 `/auth/login`

**P.S. 添加了管理员初始账户：** `admin / admin123`

### Phase 3：系统管理 ✅

**仪表盘（`DashboardView.vue`）：**
- 统计概览：用户数、章节数、文档数、对话数
- 最新动态列表

**大模型配置（`LlmConfigView.vue`）：**
- 支持 **双供应商切换**：DashScope（通义千问 Qwen3-Max）+ OpenAI 兼容（DeepSeek 等）
- 配置项：供应商、模型名、Base URL、API Key（仅写不读回显）、Temperature、Max Tokens
- 选择供应商时自动填充推荐模型名和 Base URL
- 后端对应 `llm_configs` 表 + `GET/PUT /api/v1/system/llm-config`

**智能体管理（`AgentManageView.vue` / `AgentDetailView.vue` / `AgentInvokeView.vue`）：**
- 智能体 CRUD（名称、系统提示词、模型、温度）
- 可直接在页面中调用智能体对话
- 后端对应 `agent_configs` 表 + `GET/POST/PUT/DELETE /api/v1/agents`

**系统日志（`SystemLogsView.vue`）：**
- 分页查看系统日志（筛选：级别、模块、时间范围）
- 显示操作用户名

### Phase 4：文档上传 + 解析 + 知识库 ✅

**文档解析：**
- `services/document_parse/pdf_parser.py` — PyPDF2 按页提取
- `services/document_parse/ppt_parser.py` — python-pptx 按幻灯片提取
- `services/document_parse/word_parser.py` — python-docx 按段落提取

**RAG 向量检索：**
- `services/rag/text_splitter.py` — 中文递归分块（分隔符含 `。` `；` `，`）
- `services/rag/rag_chain.py` — 嵌入生成（DashScope text-embedding-v3, 1536 维）+ Milvus 向量搜索
- 自定义 `DashScopeEmbeddings` 类封装 LangChain Embeddings 接口

**文件接口：**
- `POST /api/v1/documents/upload` — 上传文档
- `GET /api/v1/documents` — 文档列表
- `DELETE /api/v1/documents/{id}` — 删除文档

**前端页面：** `KnowledgeBaseView.vue` — 文档上传和管理界面

### Phase 5：课程问答（RAG + SSE 流式对话） ✅

**后端 SSE 流式接口：**
- `POST /api/v1/chat/sessions` — 创建对话会话
- `GET /api/v1/chat/sessions` — 对话列表
- `POST /api/v1/chat/sessions/{id}/message` — 发送消息（SSE StreamingResponse）
- `GET /api/v1/chat/sessions/{id}/messages` — 历史消息
- SSE 事件类型：`context`（知识库检索）→ `token`（流式文本）→ `recommendation`（资源推荐）→ `done`

**LLM 多供应商客户端：**
- `services/llm/dashscope_client.py` — 重构成双路径：
  - **DashScope 路径：** 使用 `dashscope.Generation.call(stream=True, incremental_output=True)`
  - **OpenAI 兼容路径（DeepSeek）：** 使用 `httpx.AsyncClient.stream("POST", ...)`
- `stream_chat()` 和 `chat()` 均支持 `provider` 参数切换

**前端 SSE 实现：**
- `utils/sse.ts` — 通用 SSE 客户端（fetch + ReadableStream 解析）
  - 支持事件类型：`token`、`context`、`recommendation`、`error`、`done`
  - 支持 `[DONE]` 结束标记
  - 支持 `AbortController` 取消请求
  - 自动携带 JWT Token
- `api/chat.ts` — 聊天 API 接口封装
- `ChatView.vue` — 完整对话界面：
  - 自动创建/加载会话
  - 历史消息加载
  - 流式逐字渲染（Markdown 渲染）
  - 加载动画（三个弹跳点）
  - 错误重试机制

### Phase 7：章节考核 ✅

完整的考核流程：配置 → 出题 → 答题 → 评分 → 报告

**后端实现：**
- 考核配置（`ChapterAssessmentConfig`）：知识点、题型分布、评价维度
- 智能出题（`question_generator.py`）：LLM 根据配置生成题目（JSON 模式）
- 自动评分（`answer_evaluator.py`）：选择/判断自动判分，简答题 LLM 评分
- 报告生成（`report_generator.py`）：维度评分 + 总体评语 + 优势/薄弱点 + 复习建议
- 接口：
  - `GET/POST /api/v1/assessment-configs` — 考核配置 CRUD
  - `POST /api/v1/assessment/start/{chapter_id}` — 开始考核
  - `GET /api/v1/assessment/{record_id}/question` — 获取题目
  - `POST /api/v1/assessment/{record_id}/answer` — 提交答案
  - `POST /api/v1/assessment/{record_id}/submit` — 交卷
  - `GET /api/v1/assessment/{record_id}/report` — 获取报告

**前端实现：**
- `api/assessment.ts` — 考核 API 封装
- `AssessmentView.vue` — 答题界面：
  - 进度条显示
  - 选择题选项 / 简答题输入
  - 逐题提交 + 全部交卷
- `AssessmentResultsView.vue` — 结果报告：
  - 总分展示（环形进度条）
  - 各维度评分（条形进度条）
  - 总体评语
  - 掌握较好 / 需要加强的方面
  - 复习建议

### 中期（后续迭代）

4. **Frontend 组件拆分**
   - 当前所有视图代码都在 `views/` 中，`components/` 目录尚无组件
   - 建议拆分为：`ChatMessage`、`ChatInput`、`ResourceRecommendationCard`、`QuestionCard`、`ReportCard`、`DocumentUploader`、`EmptyState`、`LoadingOverlay` 等可复用组件

5. **Pinia 状态管理补充**
   - 当前仅有 `auth` 和 `app` 两个 store
   - 建议补充：`chat` store（会话列表管理）、`assessment` store（考核状态管理）、`chapter` store（章节进度管理）

6. **知识库多文档关联**
   - 当前知识库支持按章节上传，但 RAG 检索未充分利用文档元数据
   - 建议：搜索结果中标注来源文档名和页码/幻灯片号

7. **资源推荐记录管理**
   - 已有 `resource_recommendations` 模型和表
   - 建议：增加推荐历史查看、收藏功能

8. **数据可视化增强**
   - 考核报告增加雷达图（ECharts 或 Canvas 绘制维度对比）
   - 仪表盘增加图表统计（学习进度趋势、考核通过率等）

9. **系统日志归档**
   - 当前日志存储在 SQLite 中，长期运行可能膨胀
   - 建议：定期自动清理或归档

10. **Milvus 连接健壮性**
    - 当前未处理 Milvus 连接失败的回退逻辑
    - 建议：Milvus 不可用时降级为纯关键字搜索或缓存搜索结果

### 技术债务

- **前端 `api/` 文件** 目前所有 API 函数都手动 `then(res => res.data)` 解包，可以统一在 `request.ts` 拦截器中处理
- **后端异常处理** 部分接口缺少统一的异常捕获，建议全局异常处理器
- **API Key 加密存储** `llm_configs` 表中的 API Key 目前是明文存储，建议加密
